As you may know, there has been a lot more talking around Polymer since they unveiled their new set of Paper Elements and the Material Design spec at Google I/O 2014 last week.

Needless to say this announcement got me really excited about Polymer & Material Design (the blog you're reading is written using Polymer), and more specifically about using Polymer & AngularJS together.


In this article, I'll be going over how you can easily leverage Polymer's <core-animated-pages> element to remove the need for ngRoute entirely, and even to replace ngAnimate if you're using it only for page transitions.
And since ngRoute & ngAnimate are now separate modules in AngularJS, this means less bytes to download for your page to load, so that's another advantage.

I will then go over how you can create an Angular directive to allow ngModel and 2-way data-binding to work on Polymer's <paper-input> element (you can see an example of it in action with the search box of this blog).
